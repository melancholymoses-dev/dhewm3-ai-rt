VK_Image_UploadCubemap allocates the staging buffer without checking VkResult for vkCreateBuffer/vkAllocateMemory/vkMapMemory. Elsewhere in the Vulkan code the VK_CHECK macro is used for these calls; here failures could lead to null handles and hard-to-debug crashes. Please use VK_CHECK (or explicit error handling + cleanup) consistently for these Vulkan API calls.

The depth image barrier uses aspectMask = DEPTH|STENCIL unconditionally. VK_FindDepthFormat can fall back to VK_FORMAT_D32_SFLOAT (no stencil), in which case including VK_IMAGE_ASPECT_STENCIL_BIT is invalid.
